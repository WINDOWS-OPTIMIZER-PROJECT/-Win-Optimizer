Domande frequenti e risposte riassunte:

1. Cosa fa esattamente OTTIMIZZATORE DI WINDOWS? "WINDOWS-OPTIMIZER"
Risposta:

Non importa quanta memoria ha un sistema: Windows è sempre installato allo stesso modo per impostazione predefinita.
Tutte le cache di Windows sono adattate ai REQUISITI MINIMI della rispettiva versione di Windows - NON viene eseguita una query hardware.
WINDOWS-OPTIMIZER adatta le diverse cache di Windows, ovvero le aree di memoria e la dimensione del file di paging in modo completamente automatico al 
hardware integrato come la memoria principale (RAM).
Questo rende Windows notevolmente più veloce e funziona anche molto più stabile a 
applicazioni complesse e molto dispendiose in termini di memoria. Ci sono una varietà di consigli per questo sul sito Microsoft 
(Fonte: https://docs.microsoft.com)
- Le formule per il calcolo delle dimensioni sono utilizzate qui esclusivamente da queste di 
Valori consigliati da Microsoft. Tutte le combinazioni per il risparmio di energia di Windows sono configurate e 
il nuovo piano di alimentazione "nascosto" "Ultimate Performance" per PC di fascia alta è sbloccato, come "Rocket Boost" 
Configurato e abilitato per disabilitare i meccanismi di risparmio energetico che influiscono sulle prestazioni.
Vengono inoltre sbloccate varie funzionalità che non sono disponibili per impostazione predefinita in tutte le versioni di Windows. 
Inoltre, varie impostazioni dell'interfaccia di Windows sono configurate in modo ottimale, le animazioni sono accelerate e alcune 
Voci aggiuntive del menu di scelta rapida come "Clean drive", "Optimize drive" facendo clic con il pulsante destro del mouse su un supporto dati 
o Impostazioni di sistema, Pannello di controllo, Tutte le attività, Modifica combinazione per il risparmio di energia o Gestione attività 
facendo clic con il pulsante destro del mouse sul desktop.
Le applicazioni possono essere avviate facendo clic con il pulsante destro del mouse con priorità diverse. 
Come ultimo punto, la maggior parte delle "ottimizzazioni" fatte da grandi "strumenti di ottimizzazione" sono annullate o impostate su 
valori sicuri e consigliati da Microsoft.

4. Cos'è l'AUTOSTART-OPTIMIZER e cosa fa? 
Risposta: 
'AUTOSTART-OPTIMIZER' è un piccolo VB-Script, che viene automaticamente trasferito alla directory di avvio automatico da 'WINDOWS-OPTIMIZER' 
di Windows. Questo piccolo strumento cerca il .exe Explorer dopo l'avvio del sistema nel task manager e assegna 
a questo processo una priorità di processo aumentata e quindi termina da solo. A causa della priorità di processo leggermente più alta,
l'intera interfaccia di Windows con tutti i menu di scelta rapida e copia / incolla / elimina accelera e reagisce in modo più diretto.

5. Posso annullare le modifiche apportate da WINDOWS-OPTIMIZER? 
Risposta:
UN RIPRISTINO DEL SISTEMA in un momento PRIMA della patch con WINDOWS-OPTIMIZER ripristinerà tutte le modifiche. 
A tal fine si puo creare un facoltativo Punto di Ripristino con il nome "Reset !c3 WIN-OPTIMIZER" - assicurarsi PRIMA dell'installazione 
che Ripristino configurazione di sistema è abilitato e funziona in modo affidabile! Inoltre, in C:, viene creato un backup completo 
del registro come "Registry-Backup.reg" prima della regolazione da parte di WINDOWS-OPTIMIZER.

13. Al 1. Riavvio dopo la patch con WINDOWS-OPTIMIZER Windows si avvia molto lentamente e funziona per minuti qualcosa in background - È normale?
Risposta: 
Sì - WINDOWS-OPTIMIZER adatta varie cosiddette cache di Windows all'hardware installato e quindi le ingrandisce drasticamente in alcuni casi. 
Queste aree di memoria devono ora essere "riempite" di informazioni dopo il primo riavvio. Il prefetching, la ricerca di Windows, 
la cache delle icone e l'indice dei file vengono ricreati e vengono attivate tutte le "attività pianificate" aperte come 
richieste di aggiornamento, controlli antivirus automatici aperti, prefetching, ottimizzazione dello sfondo del disco rigido e 
simili e viene creato un file di scambio adattato in modo ottimale al sistema, che può richiedere alcuni minuti a seconda del sistema. 
Windows sta diventando sempre più veloce nel corso delle prime sessioni. Questo è normale e avviene anche dopo un'installazione pulita di Windows. 
Il riavvio dopo l'ottimizzazione deve essere eseguito in modo tempestivo.

14. Ottimizzazione del disco opzionale - Che cos'è? 
Risposta: 
Qui lavoriamo con risorse di bordo, quindi nessun reg-cleaner o simili. Nel primo passaggio, tutte le partizioni 
vengono pulite al interno di Windows con Cleanmanager.exe e quindi deframmentate. Con gli SSD, la deframmentazione 
Defrag.exe attiva solo il Trimming(specie Deframentazione), che può prolungare significativamente la durata di un SSD,
poiché lo spazio di archiviazione inutilizzato viene rilasciato libero dove i dati sono stati eliminati.
Una corretta deframmentazione degli SSD non è possibile ed è generalmente negata da Windows.

15. Quando e con quale frequenza devo eseguire WINDOWS-OPTIMIZER sul sistema? 
Risposta: 
In linea di principio, ogni volta che l'hardware del PC come scheda madre, memoria, scheda grafica, 
CPU o scheda di rete, ecc., o i driver sono stati aggiornati per adattare in modo ottimale il sistema operativo al nuovo 
Constellation hardware + driver. Altrimenti, sempre dopo un importante aggiornamento di Windows, o secondo necessità per ripulire tutti i dischi rigidi.

16. Cos'è esattamente lo SPEED-TEST e come funziona?
Risposta:
Lo SPEED-TEST è un altro script del nostro progetto che attiva la valutazione del sistema Windows "WinSAT" e quindi produce una 
valutazione complessiva delle prestazioni del sistema e delle prestazioni di vari singoli componenti per mostrare possibili "lentamenti". 
Inoltre, il tempo della valutazione del sistema viene registrato nel test e indicato nel risultato. Questo test richiede circa 2-3 minuti, 
a seconda del sistema.

19. Windows Script host disabilitato?
Risposta: 
I due strumenti 'WIN-OPTIMIZER.vbs' e 'AUTOSTART-OPTIMIZER.vbs', nonché lo 'SPEED-TEST.vbs' e lo script 'DARK LIGHT.vbs' sono stati 
programmato nel linguaggio di programmazione Windows Visual Basic Script.
Visual Basic Script è già integrato come standard in Windows e anche attivato,
ma può essere disattivato / attivato tramite le impostazioni o vari strumenti.Se Windows Script Host 
è stato disattivato in Windows, questi script dal nostro progetto non possono funzionare e il messaggio di errore 'Windows-Script-Host 
è disabilitato!». In questo caso, eseguire il file di registro allegato >>'Windows Script Host aktivieren.reg'<< per 
riattivare il Windows Script Host (impostazione predefinita di Windows).
